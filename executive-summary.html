CAMD PhD Mentoring & Development System

A Comprehensive Solution for Student Development, Faculty Support, and Administrative Tracking

⏱️ 3-minute executive read | Optional sections below for detail

The Problem

Our doctoral program faces three critical challenges:

  • Irregular faculty-student meetings: Inconsistent advising leads to student isolation and unclear expectations
  • Poor communication: Untimely responses and unclear channels create frustration and delays
  • Milestone confusion: Students unclear about readiness criteria for qualifying exams, proposal defenses, and dissertations, leading to poor performance

Impact: 70 students and 57 faculty advisors across 6 departments struggle with communication, milestone preparation, and relationship management in our interdisciplinary program.

The Solution

An integrated 6-component system using WordPress/Formidable Forms that supports the complete doctoral journey from onboarding through graduation:

  1. Individual Development Plan (IDP): Students track goals, skills, research progress, and career planning across 5 years
  2. Lab Manual: Faculty document lab culture, expectations, and resources once for all advisees
  3. Advising Compact: Collaboratively created agreement between student and advisor establishing communication protocols, expectations, and milestone plans
  4. Summer Check-ins: Developmental reflection every August (CV/resume required)
  5. Fall Check-ins: Progress review every December
  6. Annual Review (360°): Comprehensive evaluation with student self-assessment, faculty assessment, student feedback on advising, and faculty self-reflection

Key Innovation: Forms integrate with each other—IDP data flows into Compact, check-ins reference goals, annual reviews build on previous feedback. Students and faculty work within one unified system rather than scattered documents.

Key Benefits

Stakeholder Primary Benefit
Students Clear expectations, structured development, longitudinal self-tracking, proactive career planning
Faculty Reduced administrative burden (create lab manual once), structured advising framework, visibility into student progress, 360° feedback on their mentoring
Administration Centralized tracking, early warning system for student issues, probation/dismissal flagging, compliance documentation, submission monitoring
Program Improved retention, better milestone performance, stronger advisor-student relationships, data-driven program improvements

Why This Approach Succeeds

Context: This builds on the Program Director's multi-year vision for an integrated development system. Previous approaches encountered barriers:

  • Microsoft Dataverse system: Required additional storage costs and significant development time
  • Microsoft products (Teams/Excel/Word): Too disconnected, no integration between components
  • Canvas LMS: Poor adoption—didn't match mental model of mentoring relationships, felt like "coursework"
  • Standalone Word IDP: No connection to advising or review processes, static document

This WordPress/Formidable approach addresses all previous limitations:

  • ✅ Uses existing university infrastructure (no additional costs)
  • ✅ Integrated forms with data sharing (not disconnected documents)
  • ✅ Designed specifically for developmental mentoring relationships
  • ✅ Familiar web form interface (high adoption likelihood)
  • ✅ Supports interdisciplinary research (STEM, Humanities, Practice-Based adaptations)
  • ✅ Maintains focus on student development and faculty support (administrative tracking is byproduct, not driver)

Detailed Component Descriptions Optional

1. Individual Development Plan (IDP)

Purpose: Longitudinal tracking of student goals, skills, research progress, and career planning across entire 5-year program.

Key Features:

  • Created in Year 1, updated throughout program
  • Year-by-year goal setting (Fall/Spring/Summer each year)
  • Research approach identification (STEM/Humanities/Practice-Based)
  • Milestone planning with target dates
  • Skills development tracking
  • Career pathway exploration scaffolded by year
  • CV/resume archive (uploaded with check-ins)

2. Lab Manual

Purpose: Faculty document their research culture, expectations, and resources once for all advisees.

Key Features:

  • Created once by faculty, updated annually
  • Discipline-specific templates (STEM/Humanities/Practice-Based)
  • Communication norms, meeting expectations, lab citizenship
  • Resources available (equipment, software, funding)
  • Authorship and credit policies
  • Professional development support

3. Advising Compact

Purpose: Collaboratively created agreement establishing communication protocols, expectations, and milestone plans.

Key Features:

  • Student section: Working style, career goals, support needs
  • Advisor section: Mentoring approach, availability, resources
  • Mutual agreements: Meeting schedule, response times, feedback timelines
  • Pre-populated with data from IDP and Lab Manual
  • Reviewed and updated annually
  • Both parties sign electronically

4. Summer Check-in (5 submissions over program)

Purpose: Developmental reflection every August focusing on research growth, not just task completion.

Key Features:

  • Research development questions (how thinking has evolved)
  • Discipline-specific prompts (all visible, student chooses relevant ones)
  • Creative/practical/theoretical integration
  • Career orientation (scaffolded by year)
  • Support needs identification
  • CV/resume upload (REQUIRED for summer)
  • 15-20 minute completion time

5. Fall Check-in (5 submissions over program)

Purpose: Progress review every December with forward planning for spring semester.

Key Features:

  • Same structure as Summer check-in
  • Additional: Review of last Annual Review feedback and progress
  • Spring semester planning
  • CV/resume upload (optional for fall)

6. Annual Review (360°)

Purpose: Comprehensive evaluation with mutual feedback, generating discussion guide for developmental conversation.

Four Components:

  1. Student Self-Assessment: Research progress, challenges, growth, career planning
  2. Faculty Assessment: Student progress, skills development, areas for growth, recommendations
    • Includes confidential section for probation/dismissal flagging (visible only to Academic Director)
  3. Student Feedback on Advising: What's working, what could improve, requests for changes
  4. Faculty Self-Reflection: What's working in relationship, how I could support better, commitments for year ahead

Process: Both parties complete forms → System generates discussion guide → Meeting occurs → Both sign off → Compact updated if needed

Pros, Cons, and Considerations Important

Advantages

Category Advantage
Cost Uses existing CampusPress/Formidable infrastructure. No additional budget required.
Adoption Familiar web form interface. Not associated with Canvas (previous resistance) or complex LMS navigation.
Integration Forms share data automatically. Students don't re-enter same information across forms.
Flexibility Easy to modify forms based on feedback. Not locked into rigid system architecture.
Ownership Program controls everything. Not dependent on university IT timelines or policies.
Scalability Can add more forms or features as needs evolve. System grows with program.
Data Access All submission data exportable to CSV/Excel for analysis. Academic Director has full visibility.

Challenges & Mitigation Strategies

Challenge Mitigation Strategy
User Account Management: 127 WordPress accounts to create and maintain CSV bulk import (30 minutes initial setup). Password resets handled by standard WordPress email system. Minimal ongoing maintenance.
Initial Build Time: 6 weeks to full deployment Phased approach: Build → Test → Refine → Deploy. Each form tested before moving to next.
Training Required: Students and faculty need orientation Create video tutorials and written guides. Simple interface reduces training burden. Similar to completing any web form.
Adoption Resistance: "Another system to learn" Frame as replacing scattered emails and Word docs, not adding work. Emphasize time savings (lab manual created once, IDP tracks everything in one place).
360° Review Sensitivity: Faculty receiving student feedback Frame as developmental, not evaluative. Faculty self-reflection demonstrates openness to growth. Both parties provide feedback (mutual respect).
Compliance Concerns: Are we meeting requirements? System documents all required interactions (annual reviews, milestone preparations). Creates audit trail. Probation/dismissal flagging ensures early intervention.

Areas Requiring Leadership Input

  • 360° Review Approval: Is the institution comfortable with students providing feedback on faculty advising? This is a cultural shift from one-directional evaluation.
  • Confidential Faculty Reporting: How should probation/dismissal flags be processed? Who reviews them? What's the timeline for Academic Director action?
  • Data Retention: How long should student development data be retained after graduation? University policy suggests 7 years from last attendance.
  • Success Metrics: How will we know this system is working? Suggested metrics: submission completion rates, milestone success rates, student satisfaction surveys, faculty adoption rates.
  • Mandatory vs. Optional: Should all components be required, or can students/faculty opt out of certain elements (e.g., check-ins)?
Technical Implementation Details FYI

Platform: WordPress with Formidable Forms Pro

Why Formidable Forms:

  • Already available on CampusPress installation
  • Pro version includes Views (displaying submitted data back to users)
  • Front-end editing (advisors can add to student-started forms)
  • Conditional logic (forms adapt based on research approach)
  • File uploads (CV/resume storage, max 52MB per file)
  • Email notifications (customizable to Academic Director, students, faculty)
  • Data export (CSV/Excel for analysis)

User Roles & Permissions

Role Can Access Permissions
Student Own IDP, Compact, Check-ins, Annual Review Create, edit own submissions. View own data. Cannot see other students.
Faculty Lab Manual, Advisees' forms Create Lab Manual. View/edit advisees' Compacts and Annual Reviews. Cannot see non-advisees.
Academic Director All submissions Full visibility. Receive all notifications. Export all data. Monitor completion.

Data Security & Privacy

  • All data stored on Northeastern-controlled CampusPress servers
  • WordPress user authentication required for access
  • SSL encryption for all data transmission
  • Role-based access control (users only see their own data or advisees' data)
  • Confidential faculty flags visible only to Academic Director
  • Compliant with university data retention policies

Integration Architecture

Data Flows:

  • IDP profile data → Auto-populates Compact student section
  • Lab Manual → Referenced during Compact creation
  • IDP goals → Displayed in Check-ins for reflection
  • Check-in responses → Summarized in Annual Review
  • Annual Review feedback → Referenced in next year's review
  • Compact agreements → Checked against in Check-ins (e.g., "Are you meeting as agreed?")

Mobile Accessibility

All forms fully responsive:

  • Work on phones, tablets, laptops, desktops
  • Touch-friendly interfaces
  • Progress saved automatically
  • Students can complete on any device
