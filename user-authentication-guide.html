Understanding User IDs, Roles, and Data Privacy

The Key Insight

Even though all students have the same role ("Subscriber"), each person has a unique User ID that WordPress tracks automatically. This ID is how the system knows whose data is whose.

üí° Real-World Analogy: Driver's Licenses

User Role = "Driver's License Holder"

Everyone with a driver's license has the same permissions: can operate a vehicle, must follow traffic laws, can rent cars.

User ID = Your Actual License Number

Even though millions of people have driver's licenses (same role), each license has a unique number that identifies who you are.

Same permissions, different identities = Same role, different User IDs

FAQs

Q: Do users see their User ID?

A: No. User IDs work behind the scenes. Users only see their name and data. WordPress and Formidable handle User IDs automatically.

Q: What if all students have the same role - how does it know?

A: Role determines permissions (what you CAN do). User ID determines identity (who you ARE). Different IDs = different people = different data.

Q: Can students see each other's data?

A: No. Views are filtered by User ID. Jane (ID 128) only sees entries where User ID = 128. The system automatically hides everyone else's data.

Q: How does auto-population work?

A: When Jane opens a new form, Formidable looks up: "What's the value of Field X in the IDP where User ID = current user (128)?" and automatically fills it in.

Q: How does Academic Director see everything?

A: Admin role bypasses User ID filters. Views configured for Administrator role show ALL entries regardless of User ID.

The Complete Picture

User Role = What you can do
User ID = Who you are
